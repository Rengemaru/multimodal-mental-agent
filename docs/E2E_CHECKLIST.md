# E2E 動作確認チェックリスト

**対象バージョン:** v1.0  
**確認方法:** `docker compose up` 後にブラウザで手動確認  
**記録フォーマット:** 各項目を ✅ / ❌ で記録し、❌ の場合は備考に状況を記載

---

## 事前準備

- [ ] `.env` ファイルに `GEMINI_API_KEY` を設定済み
- [ ] AivisSpeech をホスト側で起動済み (ポート 10101)
- [ ] `docker compose up --build` が正常に完了
- [ ] http://localhost:3000 にブラウザでアクセス可能
- [ ] http://localhost:8000/docs で Swagger UI が表示される

---

## フロー 1: 基本セッション (通常フロー)

### 1-1. アプリ起動確認
- [ ] http://localhost:3000 にアクセスしてアプリが表示される
- [ ] ブラウザのコンソールにエラーが出ていない
- [ ] WebSocket 接続が確立される (`Connected` 状態)

### 1-2. 自己申告スコア入力 (pre_session)
- [ ] セッション開始前に SelfReportSlider が表示される
- [ ] スライダーを動かすと値 (0.00〜1.00) が変化する
- [ ] 「送信」ボタンを押すと ACK が返り次のフェーズへ進む
- [ ] ネットワークタブで `{"type":"self_report","timing":"pre_session"}` が送信される

### 1-3. 会話 (複数ターン)
- [ ] AI からの最初の質問が ChatWindow に表示される
- [ ] AI メッセージが TTS (音声) で読み上げられる
- [ ] テキスト入力欄にメッセージを入力できる
- [ ] 送信後に StateExplanation が更新される (メンタル状態ラベル・スコアバー)
- [ ] 2〜3 ターン繰り返して会話が続く

### 1-4. 自己申告スコア入力 (post_session)
- [ ] セッション終了後に SelfReportSlider が再表示される
- [ ] 値を入力して送信できる
- [ ] ネットワークタブで `{"type":"self_report","timing":"post_session"}` が送信される

### 1-5. セッションログ保存
- [ ] 切断後に `data/sessions/` 配下に JSON ファイルが生成される
- [ ] ファイル名が `{session_id}_{timestamp}.json` 形式になっている
- [ ] JSON に `subject_id`, `consent_version`, `self_report`, `turns` が含まれる

---

## フロー 2: マルチモーダル入力

### 2-1. カメラ (表情解析)
- [ ] ブラウザがカメラ使用許可を求める
- [ ] 許可後に face-api.js が顔を検出する
- [ ] 1 秒ごとに `{"type":"face","data":{...}}` がサーバーに送信される
- [ ] カメラ拒否時はフォールバック値 (happy=0.5, neutral=0.5) が使われる

### 2-2. 音声入力
- [ ] マイクボタンを押すと録音が開始される
- [ ] 発話後にボタンを押すと録音が停止し音声データが送信される
- [ ] サーバー側で librosa による特徴量抽出が行われる
- [ ] Gemini API で音声テキスト化・感情推定が実行される

### 2-3. テキスト入力メトリクス
- [ ] テキスト入力中にキーストロークメトリクスが収集される
- [ ] 送信時に `{"type":"text","data":{interval_ms,...}}` が一緒に送信される

---

## フロー 3: デバッグモード

### 3-1. デバッグパネル表示
- [ ] URL に `?debug=true` を付けるとデバッグパネルが表示される
- [ ] 表情・音声・テキストの寄与度 (contributions) が数値で表示される
- [ ] 使用された重み (weights) が表示される
- [ ] 判定理由テキストが表示される
- [ ] `?debug=true` なしではデバッグパネルが表示されない

---

## フロー 4: エラー・異常系

### 4-1. WebSocket 切断・再接続
- [ ] バックエンドを再起動すると自動再接続される (3 秒後)
- [ ] 再接続後に新規セッションで会話を継続できる

### 4-2. AivisSpeech 未起動
- [ ] AivisSpeech が落ちていてもアプリがクラッシュしない
- [ ] TTS が失敗してもテキスト表示は正常に機能する

### 4-3. カメラ拒否
- [ ] カメラ許可を拒否してもアプリがクラッシュしない
- [ ] フォールバック値でセッションが進行する

## 既知の制限事項

- AivisSpeech はホスト側での手動起動が必要 (GUI アプリのため Docker 内で起動不可)
- face-api.js のモデルファイルは `frontend/public/models/` に手動配置が必要
- 音声入力は HTTPS 環境またはローカルホストでのみ利用可能 (MediaRecorder の制約)
